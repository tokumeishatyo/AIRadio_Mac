# S16 — DJの今日の気分 ＋ 常設コーナーのテーマ宣言リード文

> 仕様駆動ワークフロー（CLAUDE.md §7）の SoT。実装前にユーザーレビューを受ける。
> 標準13項目（要件定義 §7）の洗い出し（2026-06-14）で採用が決まった2点。参照: `s13.5-weekday-main-dj.md`（greeting / lead_in の二層化）, `../CLAUDE.md`。

## 1. 概要

標準フォーマット13項目のうち未実装だった **③「DJの今日の気分」** を採用し、あわせて**常設コーナー（フリートーク）のリード文をテーマ宣言化**する小スライス。
どちらも S13.5 で作った「冒頭コーナー＝挨拶ダイアログ／途中コーナー＝時報リード文」の二層構造に**素直に乗る**ため、変更は最小（プロンプト1箇所＋config 1行）。

- **① DJの今日の気分**: 番組冒頭の最初の会話（＝冒頭 free_talk コーナー）で、メインが挨拶・番組名・出演者紹介をしたあと、**「今日の気分」を一言**述べてから本題に入る（13項目の③を、独立コーナーではなく「OP後の会話のフリ」として実現＝ユーザー確定）。
- **② テーマ宣言リード文**: 2本目以降の常設トークのリード文を「ここからはフリートークのコーナーです」→「**ここからは〈テーマ〉について話そうと思います**」に変える（毎回同じ「フリートークのコーナー」案内をやめ、その回のテーマを宣言する）。

## 2. 配置（標準13項目との対応）

| 項目 | 現状 | 本スライス後 |
|---|---|---|
| ③ DJの今日の気分 | 冒頭コーナーは 挨拶＋番組名＋出演者紹介 のみ | 上記に続けて**今日の気分を一言**→本題 |
| ④/⑧ 常設コーナー | リード文「…フリートークのコーナーです」 | リード文「…〈テーマ〉について話そうと思います」 |

冒頭コーナー（`firstTalkIndex`）は greeting を使い lead_in を使わない（`BroadcastEngine.cornerContext` BroadcastEngine.swift:462-463）。2本目以降の talk は lead_in を使う（BroadcastEngine.swift:466-467）。よって**①は冒頭コーナー、②は冒頭以外の常設トークに効く**（重複しない）。

## 3. ① DJの今日の気分（冒頭コーナーの greeting プロンプト拡張）

- **どこ**: `DialogueScriptGenerator.makeRequest` の greeting 分岐（DialogueScriptGenerator.swift:99-100。greeting 非 nil＝冒頭コーナーのとき）。
- **変更**: 既存の「メインがまず挨拶し、番組名と本日の出演者を紹介してから本題に入る」に、**「続けて『今日の気分』を一言（今日の天気・季節・調子など）添えてから本題へ橋渡しする」**指示を加える。
  - すでに `dateContext`（季節・日付。SeasonPhrases）はプロンプトに注入されているので、気分はそれに整合した自然な一言にさせる（「今日は梅雨らしいじめじめした天気だけど〜」等）。
  - **S17（DailyContext 拡張）で曜日名・記念日が注入されると、気分の材料が自然に増える**（本スライスでは季節・日付ベース）。
- **config 化はしない**: greeting と同じくプロンプト駆動（S13.5 踏襲）。番組横断のプロンプト外部化は将来 `prompts.yaml`（[[airadio-windows-portability]] 記載）でまとめて扱う。
- 出力契約・行数下限・語尾混線禁止などの既存制約は不変。

## 4. ② テーマ宣言リード文（config のみ・コード変更なし）

- **仕組みは既存**: `CornerEngine.prepare` は lead_in の `{theme}` を、その回に選択したテーマで置換する（CornerEngine.swift:161-166。`{guest}`/`{theme}` を準備時に展開、時刻のみ run 時展開）。
- **変更は corners.yaml の free_talk `lead_in` だけ**:
  - 現状: `"{ampm}{hour}時{minute}分になりました。ここからはフリートークのコーナーです。"`
  - 変更: `"{ampm}{hour}時{minute}分になりました。ここからは{theme}について話そうと思います。"`
- テーマは `themes:` プールからランダム選択された値（CornerEngine.swift:174-177）。例: 「…ここからは旅行・おでかけについて話そうと思います。」
- **letter / guest の lead_in は対象外**（お便り・ゲストはコーナーの建付けが別。guest は既に `{guest}`/`{theme}` 入り）。
- 発話直前に時刻プレースホルダのみ展開される点は従来どおり（時報として正確、S13.5 §5）。

## 5. テスト（`swift test` グリーンが完了条件）

- **①**: `DialogueScriptGeneratorTests` に追加。
  - greeting 非 nil（冒頭）: プロンプトに「今日の気分」指示が含まれる。
  - greeting nil（途中）: 「今日の気分」指示は含まれない（冒頭のみ）。
- **②**: lead_in の `{theme}` 置換は既存機構。`CornerEngineTests`（または相当）で「free_talk の lead_in に `{theme}` を入れると、prepare 後の `PreparedCorner.leadIn` が選択テーマで置換される（時刻プレースホルダは残る）」を確認（既存の `{theme}` 置換テストがあれば free_talk ケースを足す）。
  - corners.yaml の free_talk `lead_in` に `{theme}` が含まれることを設定テストで担保してもよい（任意）。

## 6. 受け入れ基準（ライブ確認）

1. 番組冒頭のトークで、メインが挨拶・番組名・出演者紹介のあと**「今日の気分」を一言**述べてから本題に入る。
2. 2本目以降の常設トークのリード文が「**◯時◯分になりました。ここからは〈テーマ〉について話そうと思います。**」になり、毎回同じ「フリートークのコーナーです」案内が消える。
3. 既存の進行（曲・ニュース・お便り・ゲスト・特集・完全静寂）に影響なし。

## 7. 非スコープ

- DailyContext 拡張（曜日名・記念日の軽重）＝ **S17**。
- ステーション・ジャーナル（長期記憶）＝ **S18**。
- 読み正確化（VOICEVOX ユーザー辞書）／ 選曲多様化 ＝ 別スライス。
- 常設コーナー2の別テンプレ建て・今日最後の曲・リスナー投稿系・DailyTheme ＝ **不採用**（要件定義 §11）。

# S13.5 — 曜日替わりメインDJ ＋ コーナー頭の二層化（挨拶の整理・時報リード文）

## 1. 概要

2 つの要望を 1 スライスで実装する（両者は「その日のメインDJ」を軸に絡むため不可分）。

1. **曜日替わりメインDJ**: メイン DJ が日替わりで番組全体（OP・ED・時報リード文・トーク）を仕切り、
   サブ DJ が相槌・ツッコミ・応答で返す。春日部つむぎを新規追加。
2. **コーナー頭の二層化**: 「冒頭コーナーだけ挨拶、他コーナーは時報＋コーナー名」。S13 のライブ確認で
   全トークが毎回フル自己紹介から始まる問題（原因＝台本プロンプトに番組内位置・時刻が無い）の根治。

## 2. 曜日替わり編成（WeeklyCast）

`config/program.yaml` の `weekly_cast:`（曜日→DJ 並び、**先頭＝メイン**、以降サブ）。確定表:

| 曜日 | メイン | サブ | 編成 |
|---|---|---|---|
| 月 | zundamon | metan | 2人 |
| 火 | metan | tsumugi | 2人 |
| 水 | tsumugi | zundamon | 2人 |
| 木 | zundamon | metan | 2人 |
| 金 | metan | tsumugi | 2人 |
| 土 | tsumugi | zundamon | 2人 |
| 日 | zundamon | metan, tsumugi | **3人** |

- メイン = その日の cast の先頭。OP・ED・時報リード文を読み、トークを主導する。
- ニュースは曜日に関係なく **青山龍星固定**（`program.news.dj_id`、従来どおり）。
- Core `WeeklyCast`（`casts: [Int: [String]]`、Calendar の weekday 1=日…7=土）+
  `djIds(for:timeZone:) -> [String]`（その日の順序付き cast）/ `mainDjId(for:timeZone:) -> String?`。
- 乱数なし・完全決定論（テストは固定日付で検証）。`weekly_cast` 省略時は上表を既定にする。

## 3. つむぎ追加（`config/djs.yaml`）

```yaml
- id: tsumugi
  name: "春日部つむぎ"
  speaker_id: 8          # VOICEVOX /speakers で実在確認済み（ノーマル）
  persona: "一人称は「あ〜し」。ずんだもん・めたんは「◯◯先輩」、ゲストは「◯◯さん」と呼ぶ。敬語で相手に無条件の敬意を持ち、たまに現代のギャルっぽい言い回しをする。"
```

## 4. OP / ED の口上を DJ 別固定文言に

メインが日替わりになるため、OP/ED の文言・タグライン（現在ずんだもん固定の口調）を **DJ 別に持つ**。
BGM 演出（track_uri / intro / volume / ducked / outro）は**共有**のまま。`{greeting}` 等の時刻
プレースホルダは従来どおり発話直前展開。**LLM 生成ではなく固定文言**（YAML を書き換えない限り不変）。

`config/themes.yaml` 構造変更（opening / ending のみ。news は単一のまま）:
```yaml
opening:
  track_uri: "..."        # 共有 BGM 演出
  intro_seconds: 5
  volume: 100
  ducked_volume: 35
  outro_seconds: 10
  by_dj:
    zundamon:
      tagline: "ずんだもんの ケイラボAIラジオ"
      announcement: "{greeting}。{month}月{day}日、{ampm}{hour}時になりました。…お送りするのはずんだもんなのだ。…{first_song}。"
    metan:
      tagline: "四国めたんの ケイラボAIラジオ"
      announcement: "{greeting}。…お送りするのは四国めたんですわ。…{first_song}。"
    tsumugi:
      tagline: "春日部つむぎの ケイラボAIラジオ"
      announcement: "{greeting}。…お送りするのは、あ〜し、春日部つむぎでーす。…{first_song}。"
ending:
  track_uri: "..."        # tagline なし（いきなり BGM）
  intro_seconds: 5 …
  by_dj:
    zundamon: { announcement: "本日も…ありがとうございましたなのだ。…おやすみなさいなのだ。" }
    metan:    { announcement: "本日も…ありがとうございましたわ。…ごきげんよう。" }
    tsumugi:  { announcement: "本日も…ありがとうございましたー。…あ〜したちと、またね。" }
```
- Core 型: `DjSpiel { tagline: String?; announcement: String }`、
  `ThemedSegment { staging: ThemeConfig; byDj: [String: DjSpiel] }`。
  `BroadcastThemes` の opening/ending を `ThemedSegment` に変更（news は `ThemeConfig` のまま）。
- `BroadcastEngine` は OP/ED 実行時に **その日のメインの spiel** を選び、staging に tagline を載せて
  `ThemeSequencer.run`、speakerId = メイン。メインの spiel が無ければ anchor → 先頭の順でフォールバック。
- **ED は固定口上（メインが読む）で実装**。「ショートの掛け合い」は BGM 上の複数話者発話という新機構が
  必要なため本スライス対象外（将来の小改善余地として残す）。

## 5. コーナー頭の二層化

各コーナーを「**頭出しリード文（任意・時刻展開・発話直前合成）**」＋「**本編ダイアログ（LLM・事前合成）**」に分ける。

- **冒頭トーク（番組最初の talk）**: リード文なし。本編ダイアログで、メインがまず
  **時刻連動の挨拶**（`{greeting}` 由来の朝/昼/夜）→ 番組名 → 本日の出演者紹介 → 本題、の流れ。
- **その他のトーク・お便り**: 定型リード文「{ampm}{hour}時{minute}分になりました。ここからは〈コーナー名〉です。」を
  **その日のメインが読む**（ペルソナ語尾なし）。本編ダイアログは挨拶・自己紹介・番組名なしで即本題。
- **ニュース**: 既存のまま（既に時報イントロを持つ）。一貫性が取れる。

**時刻の正確さ（最重要・S8/S11 の教訓）**: リード文の時刻は再生時点で正確である必要がある。準備は
ローリングで数分〜数時間先行するため、**リード文は LLM 台本に埋め込まず・事前合成もせず**、
`CornerEngine.run` で**発話直前に時刻展開→その場で合成**する（ニュース/OP の `expand()` と同方針）。
短文（約 30 字）なので合成 1〜2 秒、曲終わりの pause 直後に小さな間が出る程度で許容。

`config/corners.yaml` にコーナー別 `lead_in:` テンプレート（時刻プレースホルダ＋自然な文言）を追加:
```yaml
- id: free_talk
  lead_in: "{ampm}{hour}時{minute}分になりました。ここからはフリートークのコーナーです。"
- id: letter
  lead_in: "{ampm}{hour}時{minute}分になりました。ここからはお便りのコーナーです。"
```
空・未指定ならリード文なし。

## 6. 主導/応答ダイナミクス（DialogueScriptGenerator）

- cast は**順序付き（先頭＝メイン）**で渡す。プロンプトに
  「進行はメイン『{main}』が主導し、他の DJ は相槌・ツッコミ・応答で返す」を常時付与。
- 位置コンテキスト（新規 `greeting: String?`）:
  - 非 nil（冒頭トーク）→「番組最初のコーナー。メインがまず『{greeting}』と挨拶し、番組名と本日の
    出演者（{names}）を紹介してから本題へ」。
  - nil（その他）→「番組の途中。挨拶・自己紹介・番組名は言わず、いきなり本題から」。

## 7. 配線（Core / Infra / App）

- **Core `CornerContext`**（新規、`prepare` に渡す）: `castDjIds: [String]`（順序付き・空なら corner.djIds）/
  `greeting: String?` / `leadIn: String?`。`CornerRunning.prepare(corner:djs:context:)` にシグネチャ変更
  （`run(corner:djs:)` 互換 extension は既定 `CornerContext()` を渡す）。
- **Core `PreparedCorner`** に `leadIn: String?` と `leadInSpeakerId: Int` を追加（run で発話直前合成）。
- **Core `CornerEngine`**: `prepare` は context.castDjIds で cast を解決（順序維持＝メイン先頭）、
  greeting を generator へ、leadIn とメイン speaker を PreparedCorner に格納。
  `run` はリード文があれば本編前に時刻展開＋合成＋再生。
- **Core `BroadcastEngine`**: 放送開始時に `blueprint.weeklyCast.djIds(for: clock.now, timeZone:)` で
  当日 cast を解決（空・未定義 id は fail-fast）。OP/ED の読み手＝メイン＋メインの spiel、
  talk の cast＝当日 cast（先頭メイン）、最初の talk セグメントにのみ greeting、他は corner.leadIn を渡す。
- **Core `ProgramBlueprint`** に `weeklyCast: WeeklyCast` を追加（`ProgramPlan` 自体は cast 非依存・不変）。
- **Infra**: `ProgramConfigLoader`（weekly_cast）、`ThemeConfigLoader`（opening/ending を by_dj 構造へ）、
  `CornersConfigLoader`（lead_in）、`DjsConfigLoader`（変更なし、YAML につむぎ追加）。
- **App `BroadcastWiring`**: themes の per-DJ 構造と weeklyCast を配線。`MenuBar` は変更なし。

## 8. テスト

- `WeeklyCast`: 全 7 曜日が確定表どおり（日曜は 3 人）/ 省略時の既定表 / timeZone 尊重。
- `ThemeConfigLoader`: opening/ending の by_dj 読み込み、news 単一維持、track_uri 欠落エラー。
- `ProgramConfigLoader`: weekly_cast 読み込み（曜日名→Int）、省略時既定、不正曜日名エラー。
- `CornersConfigLoader`: lead_in 読み込み（既定空）。
- `DialogueScriptGenerator`: 主導/応答の指示、greeting 非 nil で挨拶＋出演者紹介、nil で挨拶抑制、
  cast 先頭＝メイン名がプロンプトに入る。
- `CornerEngine`: context.castDjIds で cast 上書き、leadIn を run で発話直前に時刻展開・合成・再生
  （FakeClock で時刻確認）、冒頭は leadIn なし。
- `BroadcastEngine`: 当日 cast 解決（曜日別）、OP/ED がメインの spiel＋メイン speaker、talk cast＝当日 cast、
  最初の talk のみ greeting・他は leadIn、未定義 cast id の fail-fast、ニュースは龍星固定。
- `Fakes.FakeCornerRunner`: 新シグネチャ（context 記録）。既存テストの inline `CornerRunning` も追随。
- 既存保証（{first_song}・critical OP・キャンセル静寂・終端二重確認）は不変。

## 9. 受け入れ条件

- `swift build` / `swift test` 全グリーン。
- 曜日でメインが替わり（OP/ED/リード文/トーク主導）、サブが返す（ユーザー確認）。
- 冒頭トークのみ時刻連動の挨拶＋出演者紹介、他コーナーは「◯時◯分…〈コーナー名〉です」で始まる（ユーザー確認）。
- 時報リード文の時刻が**再生時点で正確**（ローリング準備のズレを受けない、ユーザー確認）。
- 日曜は 3 人運営（ユーザー確認）。
- エラーコード追加なし。

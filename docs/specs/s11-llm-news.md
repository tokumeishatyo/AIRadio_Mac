# S11 — ニュースの LLM 会話化（アナウンサー原稿の生成）

## 1. 概要
ニュースセグメントの原稿を「見出しの羅列 + 定型文」から、**LLM が生成する自然なアナウンサー原稿**に変える。
読み上げは引き続き青山龍星 1 人（アナウンサースタイル）。見出しをつなぐ自然な語り + 各ニュースへの
短いコメント、天気への一言を加える。

- **時報は固定**: 「時刻は{hour12}時{minute}分になりました。ニュースの時間です。」は正確性が必要なため
  LLM に渡さず固定文のまま（発話直前の時刻展開も従来どおり）。**本文だけ**を LLM 生成にする。
- **fail-tolerant**: LLM 不調時は現行の定型テンプレ原稿（S5）にそのまま倒して放送継続。
- **先行準備**: 生成には数秒かかるため、S10 のパイプラインに組み込み**放送開始時に生成開始**
  （ニュースの素材取得 + LLM 生成。デッドエアを出さない）。

## 2. スコープ

**in**:
- Core `NewsScriptGenerator`（純粋ロジック）:
  - `makeRequest(news:weather:persona:targetCharacters:styleHint:)`: 素材（見出しテキスト・天気）と
    読み手のペルソナからプロンプトを構築。出力契約 = **原稿本文のみ**（見出し・記号装飾・
    「以上です」等の締めの重複を禁止。時刻・日付は言わない = 時報と二重になるため）
  - 生成結果の整形（空行・マークダウン装飾の除去）。空なら `E-LLM-EMPTY-RESPONSE-001`
- Infra `LlmNewsScriptProvider`（`AnnouncementProviding` 準拠）:
  - news / weather の `ResearchSource` から素材を取得（個別失敗は従来どおりフォールバック文言）
  - LLM で本文生成 → 固定の時報イントロ + 本文 + 固定アウトロを組み立て
  - **LLM 失敗時は従来の `NewsWeatherProvider`（定型テンプレ）の原稿に倒す**
- `BroadcastEngine`: news セグメントも先行準備（放送開始時に `announcement()` を並行 Task で起動、
  セグメント到達時は完了済みの原稿を使う。停止時のキャンセルは S10 の機構に同乗）
- config（research.yaml に追加、いずれも省略可）:
  - `llm_script.style_hint`: 語りのスタイル指示（例: 「落ち着いた低いトーンで、簡潔に」）
  - `llm_script.target_minutes` / `chars_per_minute`: 本文の長さ目安（既定 2 分 × 320 字）
  - `llm_script.intro` / `outro`: 固定の時報イントロ / 締め（既定あり）
  - ペルソナは program.yaml の news セグメント `dj_id`（なければ anchor）の persona を使う
- App: 配線（Gemini + 素材ソース + ペルソナ解決）。`AIRADIO_DEMO=news` デモも LLM 原稿表示に更新
  （LLM キーがない環境では従来テンプレが出る = フォールバック動作の確認を兼ねる）
- テスト: プロンプト構築（素材・ペルソナ・長さ・禁止事項）/ 整形 / フォールバック（LLM 失敗 →
  テンプレ原稿）/ 組み立て（イントロ + 本文 + アウトロ）/ エンジンの news 先行準備

**out（後続）**:
- ニュースの DJ 二人会話化（アナウンサースタイルを採用したため不要。要望が出たら別スライス）
- 番組の連続ループ・13 項目拡張 / 署名 .app バンドル化 → S12 候補

## 3. 原稿の構造

```
[固定イントロ]  時刻は{hour12}時{minute}分になりました。ニュースの時間です。
[LLM 本文]      見出しを自然につないだ語り + 短いコメント + 天気への一言（約 2 分）
[固定アウトロ]  以上、ニュースと天気予報でした。
```

## 4. 受け入れ条件
- `swift build` / `swift test` 全グリーン
- `swift run AIRadioApp` → 放送のニュースが「見出しの羅列」ではなく**自然な語り + コメント**になっている
  （ユーザー確認）。時報・締めは従来どおり正確
- ニュース前のデッドエアが発生しない（先行準備）
- `config/llm.local.yaml` を一時的に外すと従来テンプレ原稿で放送が継続する（フォールバック、
  実機確認は任意 — テストで検証済みのため）
- エラーコード追加なし
